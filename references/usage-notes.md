# 信息收集与兜底规则

## 5 字段定义

| 字段 | 必填 | 示例 |
|------|------|------|
| 创作目的 | 是 | 带货 / 种草 / 涨粉 / 品牌曝光 / 引流 |
| 行业 | 是 | 厨房清洁 / 美食 / 母婴 / 美妆 / 数码 |
| 受众 | 是 | 家庭主妇 / 一线城市白领 / 宝妈 / 大学生 |
| 选题 | 是 | 农残清洗 / 一人食快手菜 / 夏日消暑 |
| 产品 | 否 | 洁娘子食用小苏打（去农残、食用级） |

## 收集流程

1. 触发技能后先判断 5 字段齐全度。
2. **必填字段齐全** → 直接进入选型生成。
3. **有缺失** → 一次性追问全部缺失项（**只问 1 轮**），话术见下。
4. **追问后用户仍不给** → 进入「自由发挥兜底」（见下）。

## 一次性追问话术模板

亲和自然、不堆术语，把缺什么一次性问清。示例（按实际缺失项调整）：

> 技能已就绪 ✅ 帮你写一条有爆款潜质的视频号脚本，方便的话补我几个信息就能开写：
>
> 1. **这次想干嘛**——带货 / 种草 / 涨粉 / 品牌曝光？
> 2. **什么行业 / 品类**？
> 3. **给谁看**——主妇 / 白领 / 宝妈 / 学生？
> 4. **聊什么选题**？
> 5. **有产品要带吗**？（不给也行，我走纯种草）
>
> 没齐也没事，你说「自由发挥」我就按爆款套路直接拟一套给你看 📝

要点：
- 一次问清，不挤牙膏；
- 明确告知「不给也行，可自由发挥」，降低用户压力；
- 不展示技术字段名（script_type / campaign_types 等不出现）。

## 自由发挥兜底规则

用户不给字段或明确让 AI 自由发挥时：

1. 参照 [viral-patterns.md](viral-patterns.md) 的「爆款潜质自检清单」，自选一套「目的×行业×受众×选题×产品」组合。
2. **必须命中自检清单 ≥3 项**，不达标重新拟定。
3. 产品缺失 → `info.product` 省略、`product_conversion=null`，走纯种草 / 涨粉向。
4. 交付时**显式告知用户**：「以下目的/行业/受众/选题为 AI 兜底拟定，可随时替换重生成」，并附上拟定组合供用户一键替换。

## 交互节奏

- 收集阶段：最多 1 轮追问，之后无论拿到与否都进入生成，不让用户反复等待。
- 生成阶段：少问多产，结构化输出直接给可用脚本。
- 交付后：脚本文档会自动**旁路提交**到服务端做登记/校验（Step 6）；提交失败不影响脚本交付，仅提示「已跳过」。之后主动询问是否需要替换字段重生成、或就某段细化改写。
